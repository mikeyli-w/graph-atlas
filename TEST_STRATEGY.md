# Graph Atlas 测试策略

本文档负责定义测试层级、关键测试场景、最低测试组合和 smoke test。本文档不负责发布路线、任务优先级或数据迁移算法；发布清单和任务优先级见 `ROADMAP_AND_BACKLOG.md`，迁移细节见 `DATA_MIGRATION_PLAN.md`。

## 1. 目标

测试策略用于保证第一阶段重构不会破坏现有原型，并验证本地知识库地基的关键行为：数据迁移、关系一致性、隐私默认值、搜索和持久化。

## 2. 测试层级

### 单元测试

优先覆盖纯函数：

- v0 nodes 到 v1 knowledge store 的迁移。
- 隐私等级到 `aiAccess` 默认值的映射。
- 从 `edges` 派生节点关系列表。
- 搜索过滤：标题、类型、标签、内容、附件名称。
- 数据校验：实体 ID 唯一、边引用存在、隐私字段补齐。
- 新增资料关联当前资料时生成可编辑的手动关系，默认关系类型和来源说明可被详情与搜索读出。

### 集成测试

覆盖关键用户路径：

- 新增资料后刷新仍存在。
- 建立关系后图谱和详情同步。
- 损坏 localStorage 后页面可恢复。
- 旧 `graph-atlas-nodes` 数据能迁移或安全回退。
- 旧版 `user` / “待关联”关系加载后会兼容归一为可编辑手动关系，并补齐关系来源对象。
- 高隐私资料默认不进入 AI 上下文预览。

### UI 回归测试

至少检查：

- 默认护照节点可见。
- 侧边栏知识域可见。
- 图谱节点可点击。
- 右侧详情面板显示基本信息、关系、附件和内容。
- 搜索后可见节点数量变化。

### 浏览器级 Smoke

使用 Playwright 覆盖真实浏览器点击链路：

- 搜索无结果后能从空状态进入收集箱新增资料。
- 加入收集箱后有明确成功反馈，并提示可在下方确认入库。
- 收集箱条目确认入库后有明确完成反馈，资料和附件都能通过页面搜索找回。
- 收集箱条目拒绝前需要二次确认，取消不会移出待整理列表且有保留反馈，确认后有明确拒绝反馈。
- 收集箱确认表单里的附件行移除前需要二次确认，取消不会丢失附件索引且有保留反馈。
- 本地文件选择能写入 IndexedDB，并在确认入库后通过详情状态和搜索结果验证。
- 超出当前本地副本上限的文件能保留索引和本地引用，并在收集箱、详情页和设置页提示外部存储待配置。
- 设置页选择本地附件目录后，超出 IndexedDB 上限的新文件能保存到本机目录，并在详情页和搜索结果中验证。
- 设置页配置可选 HTTP 上传 endpoint 后，超出 IndexedDB 上限的新文件能保存为 remote manifest，并在详情页和搜索结果中验证。
- 历史 `skipped-too-large` 附件能由用户主动补拷贝到本地目录或 remote adapter；成功时反馈包含补齐附件名，失败时不破坏原索引，没有待补大文件时会给出明确无需补拷贝反馈。
- 图谱节点拖拽后刷新页面仍保留位置。
- 顶部搜索的 `⌘K` / `Ctrl+K` 快捷键能聚焦主搜索框，搜索框内按 `Escape` 能清空当前查询，且“清除搜索”只在有查询时可用。
- 图谱悬浮缩放控件有明确语义名称，并能读出当前缩放比例。
- 恢复默认图谱布局前需要二次确认，取消不会移动当前节点且有未更改反馈，确认后能看到完成反馈。
- 布局版本能通过真实点击保存/编辑/复制/删除并看到成功反馈，默认复制标题会自动避重；确认应用、撤销回应用前位置并看到完成反馈，并在删除前二次确认；取消应用会提示当前布局未更改，取消删除会提示该版本保留。
- 详情页能通过真实点击完成节点切换、标题/类型修改并看到包含新标题和类型的保存成功反馈、Markdown 内容编辑并看到本地自动保存反馈、隐私修改并看到本地保存反馈、带明确语义、按下状态和成功反馈的收藏切换、标签新增/编辑/移除并看到具体标签反馈、来源补充/编辑/移除并看到具体来源反馈、无附件空状态补充/编辑/移除文件索引并看到具体附件反馈；取消移除标签、来源或附件时会提示对应内容仍保留；真实变更历史查看和重置确认完成反馈。
- 设置页能通过真实点击看到本地数据、隐私分布和发布准备度。
- 设置页能通过真实点击导出本地明文备份包，导出成功反馈包含下载文件名，并在二次确认重置看到取消未更改反馈和确认完成反馈后导入备份；也能导出密码加密备份，错误密码不会显示恢复确认，正确密码解密后才允许二次确认恢复；无效 JSON、非 Graph Atlas 备份或不兼容备份版本会被拒绝且不显示恢复确认，取消恢复有明确未覆盖反馈，确认恢复会替换 knowledge store 与 IndexedDB 附件副本。
- 设置页能通过真实点击看到附件存储能力诊断：IndexedDB、本地备份、inline fallback、文件系统授权状态、后端 endpoint 配置状态、历史大文件补拷贝状态；取消清除本地目录授权或后端配置时会明确提示当前配置未更改。
- 设置页能通过真实点击完成本地附件目录授权，并在二次确认后清除授权。
- 设置页能通过真实点击保存后端附件 endpoint，空 endpoint 保存有明确未启用反馈，并在二次确认后清除后端配置。
- 设置页能通过真实点击保存云同步 endpoint，测试 mock 连接，二次确认后手动推送当前资料库快照，检查远端快照差异但不覆盖本地资料库，并在二次确认后清除云同步配置；远端只有 summary 时只显示数量差异，远端包含完整 store 时显示新增/冲突预览，不兼容版本显示需检查。
- 设置页能通过真实点击看到大文件长期保存风险摘要、等待外部存储的大文件数量，并在本地附件目录或后端 endpoint 配置变化后同步更新大文件保存能力说明。
- 详情页能通过真实点击手动新增、编辑、移除并看到包含目标和关系类型的成功反馈、按方向映射双向创建关系，刷新后保留，可通过搜索找到关系依据，并能二次确认移除手动关系；取消移除手动关系时会提示关系仍保留；关系行展示可编辑/只读权限原因，无关系资料显示空状态并可从空状态建立关系；手动关系新增、编辑、移除会进入“变更历史”并显示关系类型、目标资料和来源说明；常用方向关系词会生成语义反向关系，例如“证明”对应“被证明”；关系类型输入框会复用当前资料库已有关系词，例如“计划使用”，也会提示设置页维护的本地关系模板词，例如“保管位置”，并能保存和搜索自定义关系词。
- 设置页能通过真实点击新增、移除和恢复本地关系模板词；新增模板会出现在关系类型输入提示中，移除模板不会影响已创建关系。
- 未知来源关系在详情审计中保持只读，但显示“来源待确认”和“可新增手动关系补充”的可行动说明。
- 详情页能通过真实点击确认/拒绝标签建议、确认/拒绝关系建议和确认/拒绝摘要建议，并看到包含标签、关系或摘要片段的明确完成反馈；拒绝关系或摘要建议不会新增关系或修改 Markdown。
- 引用式问答能通过真实点击展开、生成带来源草案，并在高隐私搜索范围内拒绝无依据回答。
- 移动端 360x740、390x700、390x844、430x932、768x1024 视口能看到首屏核心入口，无水平溢出，底部导航、图谱操作条和详情抽屉不重叠。
- 移动端 390x844 视口能通过底部导航进入首页、关系图、详情抽屉、收集箱和设置。
- 移动端 360x740、390x700、390x844、430x932、768x1024 视口能使用图谱操作条缩放/复位/打开当前资料，能在图谱空白区域拖动画布平移、松手惯性滑动和双指捏合缩放，复位会还原视图缩放与平移，缩放状态通过可见 `role=status` 文案同步反馈，拖拽节点刷新后保留位置，并在详情抽屉完成基础编辑和快捷编辑入口跳转；详情抽屉滚动到深处后快捷编辑条仍可见。

当前本地默认使用系统 Chrome channel（Chromium 引擎）运行，避免依赖托管 Chromium 大文件下载；CI 或新机器可使用 Playwright 托管 Chromium。

浏览器运行策略：

- 本地快速路径：`npm run test:e2e`，默认使用系统 Chrome。
- 新机器安装：`npm run test:e2e:install`，下载 Playwright 托管 Chromium。
- CI / 托管浏览器路径：`npm run test:e2e:ci`，等价于 `GRAPH_ATLAS_E2E_BROWSER=chromium playwright test`。
- 环境变量覆盖：`GRAPH_ATLAS_E2E_BROWSER=chromium npm run test:e2e` 可强制使用托管 Chromium。

GitHub Actions CI 已接入 `.github/workflows/graph-atlas-ci.yml`，会在 `graph-atlas/` 内执行 `npm ci`、`npm run test:run`、`npm run build`、`npm run test:e2e:install`、`npx playwright install-deps chromium`、`npm run test:release:preview` 和 `npm run test:e2e:ci`。本地快速路径继续使用系统 Chrome，CI 使用 Playwright 托管 Chromium，避免新机器缺少 Chrome 时漏跑浏览器 smoke。

### 隐私测试

必须覆盖：

- 高隐私实体默认 `aiAccess: false`。
- 从低隐私切到高隐私时关闭 `aiAccess`。
- 高隐私资料在 AI 上下文预览中被排除。
- UI 不显示“已加密”类误导文案。

## 3. 第一阶段最低测试组合

在未引入测试框架前，允许使用人工 smoke test + 少量纯函数验证。进入数据模型重构后，建议引入轻量测试工具，例如 Vitest。

测试引入节奏：

- P0-001 和 P0-002：允许只执行 `npm run build` + 人工 smoke test。
- 从 P0-003 开始引入 Vitest，用于覆盖数据模型、迁移和纯函数。
- 每个 P0 的最低验证门槛以 `ROADMAP_AND_BACKLOG.md` 为准，测试工具细节以本文档为准。

建议脚本：

```json
{
  "test": "vitest",
  "test:run": "vitest run",
  "test:e2e": "playwright test",
  "test:e2e:install": "playwright install chromium",
  "test:e2e:ci": "GRAPH_ATLAS_E2E_BROWSER=chromium playwright test"
}
```

优先单测对象：

- migration：v0 nodes 到 v1 knowledge store。
- privacy defaults：隐私等级到 `aiAccess` 默认值。
- relationship derivation：从 `edges` 派生图谱和详情关系。
- selectors / view models：图谱、详情、首页摘要派生。
- store validation：实体 ID、edge 引用、metadata 和 auditLog。

最低检查：

- `npm run build` 成功。
- 页面默认打开不报错。
- 损坏 localStorage 后页面仍打开。
- 新增资料能保存。
- 关系显示一致。
- 搜索“护照”能命中护照节点。

## 4. P0 测试矩阵

| P0 任务 | 主要测试类型 | 是否必须自动化 | 最低验收命令或检查 |
| --- | --- | --- | --- |
| P0-001 移除误导性加密文案 | build, smoke, UI regression | 否 | `npm run build`；人工检查 UI 不再出现“已加密”类文案 |
| P0-002 拆出 seed data | build, smoke | 否 | `npm run build`；默认护照节点、侧边栏、图谱和详情仍可用 |
| P0-003 建立 `entities + edges` 数据模型 | unit, build | 是 | 引入 Vitest 后执行 `npm run test:run`；验证 entity ID 和 edge 引用 |
| P0-004 增加 storage helper 和 repository | unit, integration | 是 | `npm run test:run`；验证空存储读取 seed store，保存后可读取 v1 store |
| P0-005 修复 localStorage 损坏回退 | unit, integration | 是 | `npm run test:run`；验证损坏 JSON 不覆盖旧数据且页面可恢复 |
| P0-006 统一图谱和详情关系来源 | unit, UI regression | 是 | `npm run test:run`；验证 graph/detail selectors 输出同源关系，并验证手动/AI 关系写入 edge-target 来源对象和关系审计文案 |
| P0-007 增加隐私字段和 AI 可见状态 | unit, privacy | 是 | `npm run test:run`；高隐私默认 `aiAccess: false` |
| P0-008 增加资料状态字段/展示 | unit, UI regression | 是 | `npm run test:run`；护照展示 AI 不可见，缺附件状态可派生 |
| P0-009 增加首页任务入口 | unit, smoke, UI regression | 是 | `npm run test:run`；人工确认第一屏包含三个任务入口 |
| P0-010 增加最小新增资料流程 | integration, smoke | 建议自动化 | `npm run test:run`；人工或集成测试确认新增资料刷新后保留 |

矩阵约束：

- P0-001 和 P0-002 可以先不引入自动化测试。
- P0-003 起，Vitest 是默认自动化验收工具。
- P0-006 起，必须覆盖 selector / view model 输出。
- P0-007 起，隐私测试属于发布红线。
- P0-010 必须确认新增资料刷新后仍保留，可用集成测试或人工回归完成。

## 6. 推荐命令

P0 阶段推荐命令：

```bash
npm run build
npm run test:run
npm run test:e2e
```

`npm run test:run` 从引入 Vitest 后成为 P0-003 及之后的默认自动化验收命令。`npm run test:e2e` 用于本地真实浏览器 smoke；CI 或新机器先执行 `npm run test:e2e:install`，再执行 `npm run test:e2e:ci`。人工 smoke test 继续用于临时视觉确认和未自动化的新交互。

- 高隐私误入 AI 上下文次数为 0。
- 图谱关系一致性错误数为 0。

## 9. 发布红线

- build 失败不能合并。
- 高隐私资料误入 AI 上下文不能合并。
- 图谱和详情关系不一致不能合并。
- 损坏 localStorage 导致白屏不能合并。
- UI 出现“已加密”类未实现承诺不能合并。
- 新增资料刷新后丢失不能合并。
