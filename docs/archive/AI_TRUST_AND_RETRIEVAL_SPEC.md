# Graph Atlas AI 信任与检索规格

## 1. AI 产品定位

Graph Atlas 第一阶段不做开放式聊天。AI 的第一职责是帮助用户查找、整理和解释资料，而且必须基于可访问资料和来源证据。

## 2. AI 可访问范围

AI 检索上下文由三个条件共同决定：

- 资料存在于本地知识库。
- `aiAccess` 为 `true`。
- 当前操作没有被隐私规则排除。

默认规则：

- 高隐私资料不进入 AI 上下文。
- 中隐私资料不进入云端 AI 上下文。
- 低隐私资料可进入 AI 上下文。

## 3. 检索结果结构

后续 AI 检索应返回：

```js
{
  query: "护照相关文件在哪里",
  results: [
    {
      id: "result-1",
      entityId: "passport",
      documentId: "doc-passport",
      sourceId: "source-passport-main",
      title: "护照信息",
      snippet: "护照首页.jpg 与签证记录.pdf",
      privacyLevel: "high",
      aiAccess: false,
      excludedReason: "高隐私资料默认不进入 AI"
    }
  ]
}
```

## 4. 回答规则

- 有来源时，回答必须展示引用。
- 没有来源时，回答必须说“知识库中没有可靠依据”。
- 被隐私排除的资料可以计数提示，但不能泄露内容。
- AI 不能直接创建、删除或修改正式节点。
- AI 建议必须进入确认队列。

## 5. 引用展示

每条回答至少包含：

- 相关节点。
- 来源标题。
- 证据片段。
- 隐私状态。
- 关系路径，如果存在。

示例：

```text
你的护照相关资料包括“护照首页.jpg”和“签证记录.pdf”。
来源：护照信息 / 附件索引。
关系路径：护照 -> 旅行清单 -> 文件资料。
```

## 6. 无证据状态

当系统没有可靠来源时：

- 不生成猜测答案。
- 展示建议动作，例如“新增资料”“检查收集箱”“扩大搜索范围”。
- 如果资料被隐私过滤排除，只显示排除数量和原因。

## 7. AI 建议确认

AI 可以建议：

- 新增节点。
- 添加标签。
- 建立关系。
- 生成摘要。
- 标记缺失资料。

AI 不可以直接：

- 删除资料。
- 降低隐私级别。
- 打开高隐私资料的 AI 访问。
- 批量修改正式图谱。

## 8. 信任体验要求

- 用户要看见 AI 使用了哪些资料。
- 用户要看见哪些资料被排除。
- 用户要能拒绝 AI 建议。
- 用户要能追溯 AI 结论来源。
