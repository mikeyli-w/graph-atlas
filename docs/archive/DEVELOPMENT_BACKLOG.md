# Graph Atlas 开发待办

## 1. 第一阶段：本地知识库地基

### P0：数据模型统一

- 从 `App.jsx` 拆出种子数据到独立数据模块。
- 定义 `entities` 数据。
- 定义 `edges` 数据。
- 定义 `documents`、`attachments`、`sources` 最小数据。
- 将现有 `baseEdges` 迁移为 `edges`。
- 将现有 `links` 改为从 `edges` 派生。

验收：

- 图谱连线和详情关系列表来自同一份 `edges`。
- 改节点标题不会破坏关系跳转。

### P0：本地存储安全读取

- 给 localStorage 读取增加 `try/catch`。
- 解析失败时回退到种子数据。
- 写入时保存完整知识库对象，而不是单一 nodes 数组。
- 为旧 `graph-atlas-nodes` 数据预留兼容迁移。

验收：

- 手动写入损坏 JSON 后页面仍能打开。
- 旧数据存在时能迁移或回退。

### P0：隐私默认值

- 将 UI 中的隐私级别映射为 `high / medium / low`。
- 新增 `aiAccess` 字段。
- 新建节点根据类型设置默认隐私。
- 搜索结果和未来 AI 上下文能按隐私过滤。

验收：

- 高隐私节点默认 `aiAccess: false`。
- UI 仍显示中文隐私标签。

### P1：基础新增资料

- 扩展“新笔记”为基础新增资料流程。
- 支持选择节点类型和隐私级别。
- 新增后写入 `entities`。
- 可选建立一条与当前节点的关系。

验收：

- 新增节点刷新后保留。
- 新增节点能在图谱中显示。
- 新增节点能出现在详情面板。

### P1：来源与附件展示

- 将现有附件迁移到 `attachments`。
- 详情面板从附件索引读取附件。
- 新增来源证据区块。
- 节点显示关联来源数量。

验收：

- 护照节点能显示原有两个附件。
- 重要节点至少显示一个来源或手动创建标记。

## 2. 第二阶段：收集箱

### P1：收集箱视图

- 启用左侧“收集箱”入口。
- 展示待整理资料列表。
- 支持新增手动资料条目。
- 支持将资料确认入库。

验收：

- 收集箱条目未确认前不进入正式图谱。
- 确认后生成实体或文档。

### P2：关系建议队列

- 支持为新资料添加建议关系。
- 建议关系需要用户确认。
- 拒绝后不写入 `edges`。

验收：

- AI 或手动建议不会自动改图谱。

## 3. 第三阶段：检索与隐私

### P1：搜索增强

- 搜索实体标题、摘要、标签、类型。
- 搜索附件名称。
- 搜索来源片段。
- 支持隐私级别过滤。

验收：

- 搜索“护照首页”能命中护照附件。
- 搜索“紧急联系人”能命中联系人和相关关系。

### P2：AI 上下文预览

- 显示当前问题或节点会带入哪些资料。
- 默认排除 `aiAccess: false` 的资料。
- 显示被排除的高隐私资料数量。

验收：

- 用户能清楚知道 AI 将看到什么。

## 4. 第四阶段：AI 问答预留

### P2：引用式回答接口预留

- 定义检索结果结构。
- 定义回答引用结构。
- 定义无证据提示状态。

验收：

- 不接 AI 也能用 mock 数据展示引用式回答 UI。

## 5. 开发顺序建议

1. 数据模型统一。
2. localStorage 安全读取。
3. 图谱和详情关系统一。
4. 隐私默认值。
5. 新增资料流程。
6. 附件与来源展示。
7. 收集箱。
8. 搜索增强。
9. AI 上下文预览。
10. 引用式 AI 问答。
