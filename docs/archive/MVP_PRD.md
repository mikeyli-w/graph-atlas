# Graph Atlas MVP PRD

## 1. MVP 定位

第一版 Graph Atlas 是一个本地隐私优先的个人重要资料与关系管理工具。它优先解决资料找不到、关系记不清、隐私不敢交给 AI、来源不可追溯的问题。

## 2. MVP 核心闭环

第一版只验收三个场景：

- 证件资料：护照、身份证、附件、来源、有效期提醒线索。
- 联系人/证明人：紧急联系人、证明联系人、与证件或工作经历的关系。
- 旅行/文件清单：旅行资料、文件集合、与证件和联系人之间的关系。

## 3. 用户故事

### 资料新增

作为用户，我希望能新增一份资料，并设置类型和隐私级别，这样我可以把重要资料放进系统。

验收：

- 标题、类型、隐私级别为必填。
- 新增后能在图谱和详情中看到。
- 刷新后仍保留。

### 关系建立

作为用户，我希望能把两个资料节点建立关系，这样我可以知道它们为什么相关。

验收：

- 关系使用 ID 存储。
- 图谱连线和详情列表一致。
- 修改标题不影响关系。

### 资料查找

作为用户，我希望搜索护照、签证或联系人时能找到相关节点和附件。

验收：

- 搜索覆盖标题、类型、标签和内容。
- 搜索可命中附件名称。
- 无结果时展示空状态和新增入口。

### 隐私控制

作为用户，我希望设置资料隐私级别，并知道 AI 是否能访问它。

验收：

- 高隐私默认 AI 不可访问。
- UI 展示中文隐私标签。
- AI 上下文预览默认排除高隐私资料。

### 来源追溯

作为用户，我希望知道某条资料或关系来自哪里。

验收：

- 重要节点展示来源证据或手动创建标记。
- 附件和来源能在详情面板看到。

## 4. 功能范围

### 必须实现

- 数据模型升级为 `entities + edges + documents + attachments + sources`。
- 本地持久化安全读取。
- ID-based 关系。
- 基础新增资料。
- 搜索与标签过滤。
- 隐私级别与 AI 可访问字段。
- 详情页展示附件和来源。

### 暂不实现

- 云同步。
- 多人协作。
- 真正加密。
- OCR。
- 向量库。
- 自动 AI 改库。
- 完整聊天式 AI。

## 5. 交互状态

必须覆盖：

- 空状态：无搜索结果、无附件、无来源。
- 错误状态：本地数据损坏、保存失败。
- 确认状态：重置、删除、入库确认。
- 编辑状态：新增资料、修改隐私、修改内容。
- 只读状态：高隐私资料进入 AI 预览时默认被排除。

## 6. 验收标准

- 当前护照默认场景完整可用。
- 图谱和详情关系列表完全一致。
- 新增节点刷新后保留。
- 损坏 localStorage 不导致页面崩溃。
- 高隐私资料默认不进入 AI 上下文。
- UI 不再误导性声称“已加密”。
