# Graph Atlas 用户旅程

## 1. 旅程目标

第一版用户旅程聚焦三个高价值闭环：证件资料、联系人/证明人、旅行/文件清单。目标是让 Graph Atlas 从“能看图谱”升级为“能完成任务”。

## 2. 核心旅程一：新增一份重要资料

### 场景

用户拿到一份新资料，例如护照扫描件、工作证明、旅行材料或联系人清单，希望先放进系统，再整理成正式知识节点。

### 用户路径

1. 进入收集箱。
2. 点击新增资料。
3. 输入标题、类型、隐私级别、摘要和附件索引。
4. 系统默认标记为“待整理”。
5. 用户确认是否生成实体节点。
6. 用户可选择与当前节点建立一条关系。
7. 确认入库后，资料出现在图谱和详情面板中。

### 成功结果

- 资料不会直接污染正式图谱。
- 资料入库后有实体、来源和隐私级别。
- 用户知道这份资料是否允许 AI 访问。

### 失败与恢复

- 如果标题为空，不允许入库。
- 如果 localStorage 损坏，回退到安全默认数据并提示本地数据异常。
- 如果用户取消确认，资料保留在收集箱。

## 3. 核心旅程二：查找护照相关文件

### 场景

用户准备出国或办理事项，需要快速找到护照、签证记录、扫描件、相关旅行清单和紧急联系人。

### 用户路径

1. 在顶部搜索输入“护照”或“签证”。
2. 系统返回相关节点、附件和关系。
3. 用户打开护照节点。
4. 右侧详情展示基本信息、附件、来源证据和关联节点。
5. 用户点击附件或关系，跳转到对应资料。

### 成功结果

- 用户能在 30 秒内找到护照相关附件。
- 用户能看到“护照 -> 旅行清单 -> 紧急联系人”的关系路径。
- 高隐私资料不会默认进入 AI 上下文。

### 失败与恢复

- 无结果时展示空状态和新增资料入口。
- 搜索命中多个节点时按相关度和最近更新排序。

## 4. 核心旅程三：准备出行清单

### 场景

用户要出行，需要确认有哪些资料已准备，哪些资料缺失。

### 用户路径

1. 打开旅行清单节点。
2. 查看关联的护照、签证记录、文件资料和紧急联系人。
3. 系统展示来源证据和附件数量。
4. 用户标记缺失资料或新增待办节点。
5. 后续 AI 可以基于低隐私资料生成检查清单。

### 成功结果

- 用户能看到出行相关资料的完整关系。
- 缺失资料可以被记录为任务。
- 系统不泄露高隐私证件内容。

## 5. 核心旅程四：建立联系人关系

### 场景

用户需要记录某个联系人是紧急联系人、项目联系人或证明人。

### 用户路径

1. 打开联系人节点。
2. 点击新增关系。
3. 选择目标节点和关系类型。
4. 填写来源或说明。
5. 保存后图谱和详情面板同步显示。

### 成功结果

- 关系使用 ID 存储，不受标题修改影响。
- 用户能看到关系来源或手动创建标记。

## 6. 核心旅程五：确认 AI 整理建议

### 场景

后续 AI 根据新资料建议新增节点、标签或关系。

### 用户路径

1. 用户在收集箱看到整理建议。
2. 系统展示建议内容、依据片段和隐私影响。
3. 用户选择确认、修改或拒绝。
4. 只有确认后的变更进入正式知识库。

### 成功结果

- AI 不会擅自修改知识库。
- 所有建议都有来源。
- 用户能撤回或拒绝建议。
