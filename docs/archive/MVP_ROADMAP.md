# Graph Atlas MVP 路线图

## 1. MVP 目标

第一版 MVP 的目标是完成“可靠本地知识库地基”，不追求完整 AI。完成后，系统应能稳定管理实体、关系、附件和来源，并为后续收集箱、检索和 AI 问答打基础。

## 2. 阶段拆分

### 阶段 0：规划资产补齐

状态：进行中。

交付物：

- `PRODUCT_SPEC.md`
- `DATA_MODEL_SPEC.md`
- `PRIVACY_MODEL.md`
- `MVP_ROADMAP.md`
- `DEVELOPMENT_BACKLOG.md`
- 更新 `PERSONAL_KNOWLEDGE_BASE_ARCHITECTURE.md`

验收标准：

- 文档能说明第一版做什么、不做什么。
- 数据模型、隐私模型和开发任务能直接指导第一阶段开发。

### 阶段 1：本地知识库地基

目标：统一数据模型和关系来源。

交付物：

- 种子数据从 `App.jsx` 拆出。
- 建立 `entities`、`edges`、`documents`、`attachments`、`sources`。
- 图谱和详情面板都从 `edges` 渲染关系。
- localStorage 安全读取和回退。
- 基础新增节点流程。

验收标准：

- 图谱连线和详情关系列表完全一致。
- 损坏 localStorage 不会导致页面崩溃。
- 新增节点刷新后仍存在。
- 当前护照示例场景可用。

### 阶段 2：收集箱与人工确认

目标：资料进入系统前可被整理和确认。

交付物：

- 收集箱视图。
- 待整理资料状态。
- 手动新增文档、附件、关系建议。
- 人工确认后入库。

验收标准：

- 新资料不会直接污染正式图谱。
- 确认后可生成实体、附件和来源。
- 拒绝的建议不会入库。

### 阶段 3：本地检索与隐私过滤

目标：先做可靠查找，再做 AI。

交付物：

- 扩展搜索范围：标题、标签、类型、摘要、来源。
- 隐私等级过滤。
- AI 可访问开关。
- AI 上下文预览。

验收标准：

- 高隐私资料默认不进入 AI 上下文。
- 搜索能跨实体、关系和附件索引命中。
- 用户能看见哪些资料将被 AI 使用。

### 阶段 4：引用式 AI 问答

目标：AI 基于允许访问的资料回答。

交付物：

- 问答入口。
- 检索结果引用。
- 无证据提示。
- 关系路径展示。

验收标准：

- AI 回答必须有来源。
- 没有来源时不编造。
- 高隐私资料默认不可用。

## 3. 优先级

最高优先级：

- 数据模型统一。
- 关系 ID 化。
- 本地持久化安全。
- 隐私默认值。

中优先级：

- 收集箱。
- 来源证据。
- 附件索引。
- 搜索增强。

后续优先级：

- AI 问答。
- GraphRAG。
- 提醒系统。
- 云同步或多端能力。

## 4. 第一阶段完成定义

第一阶段完成后，应可以说：Graph Atlas 已经不是静态 UI 原型，而是一个最小可用的本地知识库原型。

判断标准：

- 数据结构可扩展。
- 关系来源唯一。
- 本地存储稳定。
- 隐私字段有实际过滤意义。
- 后续 AI 能力有清晰接入点。
