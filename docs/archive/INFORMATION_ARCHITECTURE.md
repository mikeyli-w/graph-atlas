# Graph Atlas 信息架构

## 1. 第一版信息架构目标

Graph Atlas 的信息架构应服务任务，而不是仅服务分类。第一版将知识组织为“知识域 + 节点类型 + 关系类型 + 来源证据 + 隐私策略”。

## 2. 顶层导航

### 主导航

- 图谱：查看实体和关系。
- 收集箱：处理待整理资料。
- 搜索：查找节点、附件、来源和关系。
- 设置：管理隐私默认值和本地数据。

### 第一版侧边栏知识域

- 身份：个人资料、身份证、护照、证书。
- 工作：工作经历、证明材料、合同索引。
- 文件：文件资料、附件、扫描件。
- 人脉：联系人、紧急联系人、证明人。
- 生活：旅行清单、重要笔记、提醒。
- 项目：个人项目、项目文件、项目联系人。
- 账号：微信账号、邮箱、恢复方式索引。

## 3. 节点类型

- 证件：身份证、护照、签证、证书。
- 联系人：紧急联系人、证明人、项目联系人。
- 文件：PDF、图片、表格、压缩包、资料索引。
- 项目：个人项目、工作项目、长期计划。
- 账号：社交账号、邮箱、服务账号。
- 事件：旅行、办理事项、重要时间点。
- 笔记：说明、提醒、资料线索。
- 任务：待办、检查项、缺失资料。

## 4. 关系类型

- 属于：某资料属于某档案或知识域。
- 关联：弱关系或补充关系。
- 包含：文件夹、资料集合、清单包含某资料。
- 证明：文件或联系人可证明某经历或身份。
- 使用：账号、证件或文件被某场景使用。
- 提醒：节点触发某个提醒或待办。
- 归档：资料被归入某文件集合。
- 联系人：某人是某场景或项目的联系人。
- 依赖：某任务或资料依赖另一资料。

## 5. 标签规则

- 标签用于横向检索，不替代节点类型。
- 标签应短、稳定、可复用。
- 第一版推荐标签：证件、身份、出境、联系人、紧急、工作、证明、旅行、附件、账号、提醒。
- 系统标签和用户标签后续应区分，第一版可先统一展示。

## 6. 详情页信息层级

节点详情应按用户决策顺序展示：

1. 标题、类型、隐私级别、AI 访问状态。
2. 摘要和更新时间。
3. 关键字段和标签。
4. 关联关系。
5. 附件索引。
6. 来源证据。
7. 历史记录。
8. 原文或 Markdown 内容。

## 7. 搜索结果结构

搜索结果应分组展示：

- 节点结果。
- 附件结果。
- 来源证据。
- 关系路径。
- 待整理资料。

排序建议：

1. 标题精确匹配。
2. 最近更新。
3. 关系距离。
4. 隐私可见性。
5. 附件或来源命中。
