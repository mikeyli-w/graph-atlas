# Graph Atlas 产品指标规格

## 1. 指标目标

第一版指标用于判断 Graph Atlas 是否真的解决“重要资料找不到、关系不清、来源不可追溯、隐私不可控”的问题。

## 2. 北极星指标

**可追溯重要资料数**

定义：同时具备实体节点、至少一条关系或附件、隐私级别、来源或手动创建标记的重要资料数量。

选择原因：它衡量的不是“记录了多少笔记”，而是有多少资料真正变成了可信知识资产。

## 3. MVP 核心指标

- 资料新增数：用户新增的实体或文档数量。
- 可追溯资料比例：有来源或手动创建标记的实体占比。
- 关系完整率：有关联边的实体占比。
- 搜索成功率：搜索后用户点击结果或打开节点的比例。
- 收集箱清空率：已确认或拒绝的收集箱条目占比。
- 隐私覆盖率：有隐私级别的实体、文档和附件占比。
- AI 引用覆盖率：AI 回答中带来源引用的比例。

## 4. 质量指标

- localStorage 恢复成功率。
- 图谱关系一致性错误数。
- 无来源 AI 回答数。
- 高隐私资料误入 AI 上下文次数。
- 新增资料保存失败次数。

## 5. 第一版验收阈值

开发验收阶段建议目标：

- 图谱关系一致性错误数为 0。
- 高隐私资料误入 AI 上下文次数为 0。
- localStorage 损坏时页面可恢复。
- 默认示例数据中 100% 节点有隐私级别。
- 护照场景能在 30 秒内找到附件和关联关系。

## 6. 后续分析维度

- 按知识域：身份、工作、文件、人脉、生活、项目、账号。
- 按节点类型：证件、联系人、文件、项目、账号、事件、笔记、任务。
- 按隐私级别：高、中、低。
- 按资料状态：正式、待整理、已拒绝。
- 按来源状态：有来源、手动创建、缺来源。
